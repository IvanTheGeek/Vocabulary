# Vocabulary Map

A reference vocabulary for precise communication across design, architecture, and consulting work.
Each term is defined, scoped, and distinguished from its neighbors.

---

## Abstraction Stack

| Term | Level | Definition |
|---|---|---|
| **Paradigm** | Foundational | Unconscious worldview that shapes what questions are askable and what counts as a valid answer. You operate *from inside* a paradigm, rarely choosing it consciously. |
| **Domain** | Territory | A bounded field of knowledge and practice with its own vocabulary, rules, and patterns. Defines what problems *exist*. |
| **Layer** | Structure | A stacked level of concern within a domain or system. In systems: architectural separation. In domains: organic levels of the field itself. |
| **System** | Designed | A bounded, designed arrangement of components with defined inputs, outputs, and emergent behavior. Has a boundary; what's inside interacts by design. |
| **Model** | Representational | A composed representation of reality built from constructs. How you *think about* a system. A system *runs*; a model *represents*. You can have an incorrect model of a correct system. |
| **Context** | Situational | The active circumstances you're currently operating in — the *where* and *what*. A narrowing of domain to a specific live situation. |
| **Session** | Temporal | A bounded period of focused activity — the *when* and *how long*. The active time-bounded slice of a Context. A Context can outlast many Sessions. |
| **Construct** | Conceptual | A deliberately defined, named concept built to carve reality at a useful joint. The atomic unit of a vocabulary. |

---

## Agency Dimension

| Term | Definition |
|---|---|
| **Role** | A defined function within a system carrying responsibilities, permissions, and expected behaviors. Structural — the function, not the person. |
| **Actor** | Any entity that initiates action within a system. Exercises a Role operationally. Every Agent is an Actor; not every Actor is an Agent. |
| **Agent** | An autonomous or semi-autonomous Actor operating toward goals on behalf of a principal. Has a principal relationship, goals, and autonomy over *how* to act. |

---

## Human Causal Chain

The full chain from deep persistent drive to recorded outcome.

```
Motivation → Goal → Intent → Action → Event
                ↑
           spans Sessions
```

| Term | The question it answers | Definition |
|---|---|---|
| **Motivation** | *Why (deep)?* | Persistent underlying force or need. Doesn't complete — it's a condition. Generates Goals over time. |
| **Goal** | *What outcome?* | A defined, specific outcome spanning multiple Sessions. Completes or fails. Bridges Motivation to Intent. |
| **Intent** | *Why (now)?* | The specific, situational purpose driving a particular Action. Fleeting — lives for one action. Generated by a Goal. |
| **Action** | *What was done?* | The discrete unit of doing. Between Intent and recorded outcome. Can fail without producing an Event. |
| **Event** | *What was recorded?* | Proof that an Action completed. Immutable. The source of truth in an event-sourced system. |

---

## Mechanical Causal Chain

The chain for system-initiated and automated processes — no Motivation, no Intent.

```
Trigger → Action → Event
```

| Term | The question it answers | Definition |
|---|---|---|
| **Trigger** | *What caused it?* | A condition, signal, or event that initiates an Action mechanically or conditionally. No Intent required. |
| **Action** | *What was done?* | Same as the human chain — the discrete unit of doing. |
| **Event** | *What was recorded?* | Same as the human chain — immutable proof of completion. |

> **Note:** Action and Event appear in both chains. The chains diverge only at their origin —
> human actions originate in Motivation; mechanical actions originate in Triggers.

---

## The Two Chains Side by Side

```
Human:     Motivation → Goal → Intent → Action → Event
Mechanical:                    Trigger → Action → Event
```

When designing for Actors, design for Motivation.
When designing for Agents and automations, design for Trigger conditions.
Treating human users like trigger-response machines is a root cause of bad UX.

## Structural Rules

| Term | Definition | Distinction |
|---|---|---|
| **Constraint** | A rule limiting valid states or actions. Gates entry — a pre-condition. | You violate a constraint by *attempting* a forbidden action. |
| **Invariant** | A condition that must always hold regardless of what actions occur. Perpetual integrity. | An invariant violation means the system is in a *broken state* — not just that something was attempted. |

---

## Analytical Tools

| Term | Definition |
|---|---|
| **Lens** | A deliberately chosen analytical filter applied to examine something. Composable, switchable, conscious. You can apply multiple lenses to the same thing sequentially. |
| **Assumption** | An unstated or stated belief underlying reasoning, design, or communication. Invisible when right; breaks things when wrong. Lives underneath every other term. |

> **The triad:** Paradigm → Perspective → Lens
> — from *unconscious worldview* to *positional viewpoint* to *deliberate analytical tool*.

---

## Language

| Term | Definition |
|---|---|
| **Vocabulary** | The committed set of constructs for a domain or system, with agreed-upon meanings. Not just a dictionary — the act of committing to consistent use. |
| **Interface** | The defined crossing point between two systems, components, actors, or layers. Specifies what can be exchanged and how, without exposing internals. Boundary says "here's the edge"; Interface says "here's the door." |

---

## Communication Chain

```
Perspective → Framing → Audience
```

| Term | Definition |
|---|---|
| **Perspective** | How you see from where you stand. Shaped by Role and Paradigm. Can be unconscious. |
| **Framing** | The deliberate act of structuring how something is presented to shape how it's received. The bridge between Perspective and Audience. Framing doesn't change the underlying construct — it changes what gets foregrounded. |
| **Audience** | The specific people who will receive, use, or be affected by the output. Shapes form and language, not underlying content. |

## Quick Reference: Adjacent Term Distinctions

| If you're reaching for... | Make sure it isn't... | Distinction |
|---|---|---|
| Paradigm | Perspective | Paradigm is unconscious worldview; Perspective is a positional viewpoint you can name |
| Perspective | Lens | Perspective is where you stand; Lens is what you deliberately pick up to analyze with |
| Domain | System | Domain is territory; System is a designed structure operating within it |
| System | Model | System runs; Model represents |
| Context | Session | Context is where/what; Session is when/how long |
| Construct | Vocabulary | Construct is one named concept; Vocabulary is the committed collection |
| Constraint | Invariant | Constraint gates entry; Invariant defines perpetual integrity |
| Role | Actor | Role is structural function; Actor is the entity exercising that function |
| Actor | Agent | Actor initiates; Agent initiates *autonomously toward goals* |
| Motivation | Intent | Motivation persists; Intent is situational and fleeting |
| Goal | Intent | Goal spans Sessions and completes; Intent lives for one action |
| Trigger | Intent | Trigger is mechanical cause; Intent is purposeful human cause |
| Framing | Perspective | Perspective is how you see; Framing is how you package it for an audience |
| Interface | Boundary | Boundary defines the edge; Interface defines what crosses it and how |
